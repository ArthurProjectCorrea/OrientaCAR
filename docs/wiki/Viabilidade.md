## A solução é viável nos três aspectos

### Legal

O app **não substitui o SICAR** — ele prepara o produtor para usar o sistema oficial.

- Toda a operação está dentro do previsto pela **Lei 12.651/2012** (Código Florestal) e pelo **Decreto 7.830/2012**
- As bases de dados utilizadas (SICAR, MapBiomas, PRODES, Gov.br) têm **acesso público**
- Não há coleta ou processamento de dados fora do que o produtor já declara ao governo

### Tecnológico

Toda a stack é baseada em **tecnologias abertas e gratuitas**.

Um protótipo funcional foi desenvolvido em **48 horas** no haCARthon:

| Tecnologia | Uso |
|---|---|
| React 19 | Interface do usuário |
| TanStack Router | Navegação entre telas |
| Tailwind CSS | Estilo e responsividade mobile |
| Leaflet / turf.js | Mapas e cálculo de área |
| IA (LLM) | Assistente contextual |
| PWA | Distribuição sem loja de apps |

### Operacional

- O app funciona como **PWA no navegador do celular** — sem necessidade de publicação em loja de apps
- O conteúdo do assistente pode ser atualizado **sem programação** por equipe técnica do órgão
- Distribuição pode ser feita via **QR code** nos órgãos ambientais estaduais e sindicatos rurais

## O que seria necessário para a versão de produção

| Recurso | Detalhe |
|---|---|
| Especialistas em legislação | Validar o conteúdo do assistente de IA com base no Código Florestal |
| Analistas das OEMAs | Mapear os erros mais frequentes nos cadastros e calibrar o módulo de análise |
| Produtores rurais | Testes de usabilidade com diferentes perfis e níveis de familiaridade digital |
| Parceiros institucionais | Serviço Florestal Brasileiro, INCRA, secretarias de meio ambiente estaduais |
| Equipe de desenvolvimento | 1 frontend, 1 backend, 1 especialista em legislação, 1 designer |
| Investimento inicial | **R$ 300 mil** para versão de produção e primeiros 12 meses no Tocantins |
